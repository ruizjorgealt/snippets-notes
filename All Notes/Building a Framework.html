My Automation Test Rules

  • NEVER require the tests to declare variables.
  • NEVER require the tests to use the new keyword or create new objects.
  • NEVER require the tests to manage state on their own. (Exceptions might occur)
  • NEVER expose the browser or DOM (Document Object Model) to the tests or let them manipulate it directly.
  • ALWAYS reduce the number of parameters for API calls when possible.
  • ALAWAYS use default values instead of requiring a parameter when possible.
  • PREFER to make the API easier to use over making the internals of the API less complex.
  • PREFER using enumations and constants when requiring the test to pass in primitive types. (Make the input bounded when possible)
