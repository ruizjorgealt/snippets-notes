Options tell a command how to operate

  • Start with a dash
  • Usually one letter
  • More than one offered by most commands
  • Options can be combined with a single dash
    • ​ls -lah ~/Desktop​

Create and remove folders

The mkdir allows us to create folders and of course, directories.

The following command created a simple folder called files in the desktop folder
mkdir Desktop/files

The argument -p allows us to create the necessary folders when creating a directory that might have some missing folders.
For example: ​mkdir -p Desktop/files/downloads/music​
Notice that the folders files and downloads might not exists; however, the argument -p makes sure they are created.

To delete a folder we use rmdir; however, it is important to understand that the folder must be empty. 
For example
rmdir Desktop/files
The previous example would not work because it’s not empty, it contains some folders inside.

Copy, move, and delete files and folders

To copy a file in the same directory
cp my_text.txt my_text_copy.txt

To copy a file to another directory
cp my_text.txt ~/Desktop/

To copy a file to another directory with a different name
cp my_text.txt ~/Desktop/my_text_copy.txt

Moving files allows us to move and rename files within one command
mv my_text.txt files/my_text_copy.txt

Wildcards

* – Any number of any characters
? – One of any character

The following command moves all of the files with the .txt extension to a particular folder
mv *.txt Documents/text_files

The following command moves all of the files in a folder to a particular folder
mv Documents/myFiles/* Document/myOldFiles​

In order to use the ? wildcard, make the assumption that we have a folder with the following files:
poems.txt
poems3.txt
poems4.txt

The ? wirldcard will allow us to delete the other poems files that are not poems.txt
rm poems?.txt

Deletion

To delete various files within one folder
​rm -r files/​

To delete single files
​rm files/my_text.txt​

Find Files from the command line

​find . -name “poe*”​
The previous command finds in the current directory and its children all files with a name starting with poe

​find ~/Documents -name “*d*”​
Find in the documents folder all files/folders with a name containing the letter d

Understand user roles and sudo

user – No changes to the system
root – Can make any changes to the system

sudo – super user do | Grants users root access

For example, in order to a normal user to view the /root folder, it is appropriate to type in:
sudo ls root/

To give up the root priviledges
sudo -k​

To change to root user
su root

File Permissions

rwxrwxrwx the_file
  • user
  • group
  • others
r – run
w – write
x – execute

We can change the permissions of a file using the chmod command.

Two methods to represent permissions
  • Octal
    • 755
    • 644
    • 777
  • Symbolic
    • a=r
    • g+w
    • o-x
chown root text.txt ← Change ownership to root

Octal File Permissions

Read (4) Write (2) Execute (1) Result
User r w x 7
Group r - x 5
Others r - - 4

Symbolic Value Changes

Original Symoblic Value Result
rw-e--e-- +x rwxr-xr-x
rwxrwxrwx g=w,o=r rwx-w-r--
rwxrwxrwx a-x rw-rw-rw
a – All
u – User
g – Group
O – Others
The following command would remove the execute permissions from the user
chmod u-r ~/Desktop/text.txt 
cat ~/Desktop/text.txt
