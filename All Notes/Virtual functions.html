Virtual functions

Virtual Functions

When we use virtual functions, we are telling C++ that we want to call derived functions through a base class.
#include <iostream>
using namespace std;
class Enemy{
public:
virtual void attack(){}
};
class Ninja: public Enemy{
public:
void attack(){
cout << "Ninja attack!" << endl;
}
};
class Monster: public Enemy{
public:
void attack(){
cout << "Monster attack!" << endl;
}
};
int main(int argc, char *argv[]) {
Ninja n;
Monster m;
Enemy *enemy1 = &n;
Enemy *enemy2 = &m;
enemy1->attack();
enemy2->attack();
return 0;
}
We are basically executing different functions of the same type.

Abstract base classes and pure virtual functions

Abstract classes and pure virtual functions can be used to define an interface that must be implemented by derived classes. In other words, whenever you write a pure virtual function, you need to overwrite it.
#include <iostream>
using namespace std;
class Enemy{
public:
virtual void attack()=0;
};
class Ninja: public Enemy{
public:
void attack(){
cout << "Ninja attack!" << endl;
}
};
class Monster: public Enemy{
public:
void attack(){
cout << "Monster attack!" << endl;
}
};
int main(int argc, char *argv[]) {
Ninja n;
Monster m;
Enemy *enemy1 = &n;
Enemy *enemy2 = &m;
enemy1->attack();
enemy2->attack();
return 0;
}
An abstract class is a class with nothing but pure virtual functions.
