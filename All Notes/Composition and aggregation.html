Composition and aggregation

Class Composition

Class composition occurs when an object of one class owns an object of another class. Composition is also known as “IS A” relationship.

Composition describes a “IS A” relationship:
Document is many pages.
Society is many people.

If one were to delete a document, than all of the associated pages should be deleted too. ← Implies ownership

Simple composition example:
​class Car{
  private:
    Motor* motor;
  public:
    Car(){motor = new Motor();}
    ~Car(){delete motor;}
};

Class aggregation

Aggregregation is quite similar to composition; however, in this case we are dealing with pointers. This is when an object of one class owns an object of another class (requires pointers). This is also known as a “HAS A” relationship.

Aggregation describes a “HAS A” relationship:
A customer has a address.
A car has a engine.
A bank has many bank accounts.
A university has many students.


One of the main differences between composition and aggregation is the lifetime of the object.

Simple aggregation example:
​class Department{
  private:
    Professor* professor;
  public:
    Department(Professor* p) : professor(p){}
};
