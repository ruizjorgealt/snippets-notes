Traffic Capture and Analysis

Network administrators use a packet sniffer, network monitor, or network analyzer to monitor and troubleshoot network traffic.

The sniffer captures the packet and displays its values.
Uses for Pakcet Analysis
  • Analyze network problems
  • Detect network intrusion attempts and network miuse
  • Perform regulatory compliance
  • Monitor bandwidth utilization
  • Verify endpoint security status
Common Packet Analyzers
  • Cain and Abel – Recovers passwords
  • NarusInsight – Monitors all internet traffic
  • dSniff – Monitors a network for interesting traffic
  • Ettercap – Intercept traffic on a network segment
  • Tcpdump – Runs from the command-line
Placement is Key
  • All traffic is not created equal
    • Depending on placement, you may only capture a portion of the total network traffic.
    • Traffic may be unicast, broadcast, or multicast.
  • To see all traffic on a switch
    • Use Port Monitoring or SPAN
    • Use a full duplex tap in line with traffic 

Managing Capture Options

It is important to go to capture options in order to specify the type of connections you wish to capture; however, this is not recommended when troubleshooting because you could potentially miss important information.

Using Display and Capture Filters

The expression builder allows you to make granular searches of the packets of you have captured. 

Perhaps you are looking for TCP packets whose flags FIN are set to true.

We can use combined conditions in the expression to filter very specific packets.
