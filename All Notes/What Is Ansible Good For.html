What Is Ansible Good For?

Configuration Automation
  • Hand management is too slow or prone to error
  • Application dveelopers have defined environment expectations
  • Ansible provides simple language for expressing configuration directives 
    • Readable by both developers and operators
Infrastructure Provisioning
  • Significant delays
  • Hand-managed resources
  • Annual budgets
On-Demand Infrastructure
DevOps
  • Development and operations – Work closely each other
Experiments
  • Starts quickly
  • Fail quickly
  • Fast feedback
Infrastructure as Code
  • Lives with application configuration code
Modules can create infrastructure and assert the configuration of the infrastructure.
Ansible in the Modern Era
Provides modules to create infrastructure, as well as modules to assert the configuration on that infrastructure.
Orchestration – Set of coordinated actions
  • Complicated actions
  • With specfic order
Delegation
  • Redirect connection
  • Keep context of original target
Configuration – Manage systems configuration
  • Initial application install
  • Future changes to application configuration
Handlers – Will perform when requested
  • Reacts to task changes via notify
  • Allows a single reaction to multiple changing tasks
– Ansible gives the ability to CREATE and MANAGE infrastructure.
• Ideally we would have two playbooks, one that creates the actual infrastructure and another playbook that would configure it. ← Provisioning
Loops – Task control that directs Ansible to repeat a task iterating over a set of things.

Variable data is updated per loop for use in conditional or module arguments or templates.
There are many types of Loops available:
  1. Variable sets
  2. Sequences
  3. Retries on failure
Example: Have a playbook that creates docker customized docker containers.
Ansible provides a method for ad hoc operation: 
Ad hoc execution examples:
Rebooting all web servers
ansible -i webapp web -m debug -a "msg='shut -r now'"
Querying to see which version of Ansible is installed:
ansible -i webapp web -m command -a "rmp -q ansible"
Quickly copying a file:
ansible -i webapp web -m copy -a "src=critfile dest=/tmp/critfile"
Forks – Maximum number of concurrent hosts that Ansible will connect to at the same time for a task.
  • Limited by the size of play or task targets
  • Further limited by serial batch size
ansible -i hosts host1 -m command -a "cat /etc/*-release"
