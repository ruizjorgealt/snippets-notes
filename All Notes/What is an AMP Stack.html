What is an AMP Stack?

AMP stacks are used for multitier web applications; they are built using multiple software products or tiers.
There are four major tiers in a web application:
  • Client: –Web browser
    • Sends requests to the server and render HTML, images, and CSS
    • Execute client-side code such as JavaScript
  • Web: –HTTP server (Apache)
    • Receives requests from the client and returns responses.
    • HTTP server dispatches requests to the business tier.
    • HTTP Server (Apache, IIS, etc.)
  • Business: –Application server (PHP)
    • Application Server (PHP, ASP.NET, Node.js, Java EE, etc.)
    • Interacts with the data tier or the database server.
  • Data: –Database server (MySQL)
    • Handles operations dealing directly with data.

Interactions

(A) Request is made: Client → Apache → PHP → MySQL
(B) Request is fulfilled: Client ← Apache ← PHP ← MySQL
In a multitier environment, most of these components don’t know about each others existence.
For example: The client doesn’t know about MySQL, and MySQL doesn’t know about Apache.
In a production environment, the database tends to be hosted on a separate server.

Standard Ports

A port is a numeric value. Each server application is listening on a port and on the client side each browser is sending requests on a port. There can only be one single application listening on each port.
HTTP: Port 80
HTTPS: Port 443
MySQL: Port 3306

PHP Installation

PHP installation requires modules that will allow Apache to connect to PHP and MySQL to connect to PHP.
Apache ←module→ PHP ←module→ MySQL
Modules:
  • libapache2-mod-php
  • php-mcrypt
  • php-mysql
PHP is installed with the configuration for Apache automatically.
It is also recommended to turn on display_errors setting for PHP.
Command:
apt-get install apache2 mysql-server php libapache2-mod-php php-mcrypt php-mysql
