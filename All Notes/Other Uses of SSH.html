Other Uses of SSH

SSH File Transfer Protocol (SFTP)

  • Encrypted
  • One port
  • Well-supported
  • FileZille
    • Windows
    • MacOS
    • Linux
The idea of FTP is quite simple, you need to provide the same credentials as if you were using openning a connection using SSH.

Credentials needed:
Host
User
Password
We can also use FTP through MacOS terminal using the sftp command.

Transferring files with secure copy (SCP)

  • Comes with MacOS and Linux
  • Windows version: PSCP, available from putty.org
This command allows us to transfer files from one host to another.
For example, the following command sends a jpeg file from the Linux Mint virtual machine to my iMac:
cd ~/Pictures
scp art4.jpeg jorgearuiz@192.168.0.6:/var/Documents
Password:
art4.jpeg  100% 1763KB 1.7MB/s 00:00
To copy from the an external host to your local host, simple switch the order of the commands.
For example:
scp scott@orion.dev:art1.jpg ~/Desktop/files/

Creating an SSH tunnel

Port forwarding via SSH (SSH tunneling) creates a secure connection between a local computer and a remote machine through which services can be relayed. Because the connection is encrypted, SSH tunneling is useful for transmitting information that uses an unencrypted protocol, such as IMAP, VNC, or IRC.

SSH is remote login shell that helps you to connect remote machines using encrypted connection. So once you made ssh connection to any remote host the connection between hosts are secure and encrypted.

SSH tunneling is routing your traffic through SSH secure connection.

In simple words SSH tunneling is nothing but one connection is encapsulated by another connection. By taking this as a advantage we make tunnels by using SSH client.
