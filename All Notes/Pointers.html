Pointers

Pointers

Pointers and the address operator: Every variable is assigned a memory location whose address can be retrived using the address operator → &.
The operator ​&​ can be used to retrieve the address information of any variable, remember that each byte of memory has a unique address. Therefore, by placing the address operator ​&​ we can retrieve the variable’s address on memory.

Here is how to print and maybe store the memory address of a variable.
​cout << float(&var);​
Float must be used to prevent the lost of data. If the float keyword is not present, then we will get the hexadecimal value.

Unfortunately, running the previously recommended piece of code, gives us errors. Therefore, as of now, it is recommended to simply access the address of a variable by using the address operator.

Pointer variables

A pointer variable is a container that holds the address of memory locations. A variable that holds the address is called a pointer variable or simply a pointer.

The asterisk ​*​ states that the variable is a pointer, and a pointer would be declared in the following way.
​int *Ptr;​ 
​int num = 25;
​Ptr = &num;​
We can change the value of the variable by simply changing the value of the pointer.
#include <iostream>
using namespace std;
int main(int argc, char *argv[]) {
int num = 10;
int *Ptr;
Ptr = &num;
*Ptr *= 2;
//BOTH DISPLAY 20
cout << num << ' ' << *Ptr;
return 0;
}
The asterisk before the pointer simply allows us to access the variable’s value and not the address. However, the address must always be stored before the programmer wants to access the data.

Relationship between arrays and pointers

Arrays can be used as pointer constants, and pointer constants can be used as arrays.
#include <iostream>
using namespace std;
int main(int argc, char *argv[]) {
int var[5] = {1,2,3,4,5};
for(int i=0; i<5; i++){
cout << *(var+i) << ' ';
}
return 0;
}
Assigning the address of an array to a pointer:
We simply assign the array to the pointer like this
​Ptr = varArray;​
Then the pointer can access the elements of the array by using a subscript.
Take a look at the following program for a better understanding.

Notice that once the pointer has received the address of an array, we can easily access those elements through a subscript.

Initializing pointers

Pointers may be initialized with the address of an existing object. However, the pointer must be of the same data type as the variable’s address.

Legal ways of initializing a pointer:
​int myValue;
int *varPointer;
varPointer = &myValue;​

Initializing with an array:
​int varArray[SIZE];
int *Ptr;
Ptr = &varArray;

A null operator:
​int *Ptr = NULL;​
It is important to assign the value of ​NULL​ in order to prevent the pointer from addressing another variable that we might not intent to modify.

Relational operators

!=

In order to avoid comparing a pointer’s memory address, such as:
​&var1[i] == &var2[i]​
Use the asterisk ​*​ operator in order to compare values, for example:
​*var1 == *var2​ ← SINGLE NON-ARRAY POINTER VARIABLES REQUIRE AN ASTERISK!!!
However, we do not need the asterisk when comparing array elements/values through pointers:
var1[i] == var2[i]​
​
POINTER VARIABLES THAT DO NOT HOLD AN ARRAY ADDRESS ARE REQUIRED TO HAVE AN ASTERISK IN ORDER TO ACCESS THE ACTUAL VALUE.

Function as function parameters

A pointer can be used as a function parameter. It gives the function access to the original argument, much like passing by reference. It is way easier to simply pass a reference variable than a pointer.
Function call:
​displayFunction(&Ptr);​
The prototype:
​displayFunction(int *);​
Function header:
​void displayFunction(int *Ptr){…}​

Remember that you should use an address operator → ​&​ if you are passing to a ​**​, else you will have problems. This is known as pointer to a pointer.

Notice that when sending pointers to functions, we are simply sending the address of the variable and not a pointer variable.
​int i;​ ← Declaring simple variable
getNum(&i);​ ← Sending the address in function call
void getNum(int *num){…}​ ← Receiving the address in a pointer variable

Double pointer

Also known as pointer to point or double indirection, is a technique in which a pointer points at another pointer, which points at a variable.
#include <iostream>
void display(int **);
using namespace std;
int main(int argc, char *argv[]) {
int i = 10;
int *Ptr = &i;
display(&Ptr);
return 0;
}
//FUNCTION RECEIVING ADDRESS OF POINTER.
//WE KNOW HAVE A DOUBLE POINTER OR DOUBLE INDIRECTION
void display(int **i){
cout << "Pointer's pointer address (i): " << i << endl;
cout << "Pointer's address (*i): " << *i << endl;
cout << "Pointer's actual value (**i): " << **i;
}

Pointers to constants and constants to pointers

A pointer to constant may not be used to change the value it points to; a constant pointer may not be changed after it has been initialized. Even if using the asterisk operator ​*​ you will not be allowed to change the elements of the array.
Declaring the constant array:
​const int varArray[SIZE];​
Calling the function:
​display(varArray);​ ← Notice the lack of address operator because it’s an array.
Function header:
​void display(int const *Ptr){…}
#include <iostream>
void display(int *const, int);
using namespace std;
int main(int argc, char *argv[]) {
const int SIZE = 3;
int n[SIZE] = {1,2,3};
display(n, SIZE);
return 0;
}
//RECEIVING ARGUMENT INTO A CONSTANT POINTER
/********************************************
* REMEMBER THAT WE HAVE PASSED AN ARRAY TO *
* TO A CONSTANT POINTER VARIABLE *
********************************************/
void display(int *const n, int SIZE){
for(int i=0; i<SIZE; i++){
cout << n[i] << endl;
}
}

Constant pointers

Notice that it will not let you change the address of the pointer; however, you can still change the value of the variable the pointer is pointing to.
​int* const Ptr = &var;​
#include <iostream>
using namespace std;
int main(int argc, char *argv[]) {
int i = 0;
//ASSIGNING ADDRESS TO CONST POINTER
int* const n = &i;
//MODIFYING VALUE BUT NOT ADDRESS
*n += 10;
//DISPLAYS MODIFIED VALUES
cout << i;
return 0;
}
Trying to modify ​n​ would be illegal.

Pointers to class objects and structures

Pointers and dynamic memory allocation can be used with class objects. Declaring a pointer to a class is like declaring a pointer to any other data type.

​Class Cube{
  int w;
  int h;
};

Declaring object and pointer:
​Cube *pCube;
Cube rect;
Assigning the address:
​pCube = &rect;​
Assigning values to the member variables:
​(*pCube).w
(*pCube).h​
Accessing member variables values:
​pCube->w
pCube->h​
Accessing a member function:
​pCube->getArea()​

Pointers to class objects as object parameters

Declaring the function prototype, calling the function, and constructing the function header is quite simple, for it is handled as if it was any other type of data.
#include <iostream>
using namespace std;
//CLASS
class Cube{
public:
int h;
int w;
};
//FUNCTION PROTOTYPE
void display(Cube *);
int main(int argc, char *argv[]) {
Cube c;
//SENDING THE ADDRESS ONLY
display(&c);
return 0;
}
//RECEIVING THE ADDRESS IN A POINTER VARIABLE
void display(Cube *c){
cout << "Enter height: ";
cin >> (*c).h;
cout << "Enter width: ";
cin >> (*c).w;
cout << "Area: " << c->h * c->w;
}
