Connecting Securily with a Key

Using a Key for more-secure access

  • More secure than Password access
  • Key pair: Public key and Private key
  • Server holds the Public key, you keep the Private key

One or Many Keys?

  • One key for each computer: more setup, more resilient 
  • One key for all computers: less setup, single point of failure

Generating a key on Mac OS X or Linux

To generate a key pair on a Mac or Linux machine, in the terminal you need to type ssh-keygen -t rsa to specify the RSA encryption algorithm type. RSA is a widely used encryption algorithm. This will start up an interactive series of prompts asking you where you want to store the key.

This will generate: ← Use ls .ssh
id_ra ← Private key | Stays on our computers we will use to connect to the main server
id_rsa.pub ← Public key | Uploaded to the server which wish to connect via SSH

To get the public key file onto the server:  – The following command copies the private key to the server that we which to connect to using our public key.
On linux → ssh-copy-id user@example.com
On mac → scp .ssh/id-rsa.pub scott@orin.dev:
scp – Secure copy

If this is the first time setting up a key on your server, you'll need to create the .ssh folder in your home folder. You can do that mkdir.ssh and then concatenate the key into the authorized keys file. All right, cat id_rsa.pub > .ssh/authorized_keys and press enter. And that's it, now I'll type exit to disconnect.

Connecting to an SSH server from MacOS and Linux using a key

To connect using the private key.
ssh -i .ssh/id_ra scott@orion.dev

To create a short open a text and save it to your .ssh folder
Then you can ssh into that file
ssh orion
