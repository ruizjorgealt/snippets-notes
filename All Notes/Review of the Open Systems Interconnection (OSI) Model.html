Review of the Open Systems Interconnection (OSI) Model

OSI Model – Open Systems Interconnection Model

  • Seven layer model
    • Each layers provides services to the next layer
  • Common framework for better understanding
  • Provide interoperability

PDU – Protocol Data Unit

Helps us describe and understand how each layer breaks down data to prepare it for transmission. It also defines what shape the data is in as it is passed down the layers.

Layer 7: Application

  • Initiates contact with the network
  • HTTP, FTP, SMTP
  • The PDU is DATA

Layer 6: Presentation

  • Formatting
    • Use the proper file format
  • Encryption (Optional)
  • Compression (Optional)
  • The PDU is DATA

Layer 5: Session

  • Set up, maintain, and tear down a session
  • DNS works in this layer – Requires proper IP address
  • The PDU is DATA

Layer 4: Transport

  • Transportation of DATA
  • TCP ad UDP
    • TCP – End-to-end reliability
    • UDP – Not reliable
  • PDU is SEGMENT
  • PORT address – What associate us with the application
    • HTTP uses port 80

Layer 3: Network

  • Addressing and Routing
  • IP, ARP, and ICMP
  • The PDU is PACKET
  • IP address (logical addressing)

Layer 2: Data Link

  • Frame formation
  • Ethernet II
  • The PDU is FRAME
  • MAC address (physical addressing)

Layer 1: Physical

  • Media signal
  • Binary transmission
  • The PDU is BITS
